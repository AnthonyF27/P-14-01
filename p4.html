Funciones y Procedimientos

Fecha: 10/01/2024

Las funciones y los procedimientos son conjuntos de instrucciones que realizan una tarea en particular y permiten crear programas complejos, mediante un reparto de tareas que permite construir el programa de forma estructurada y modular.


Funciones

Las funciones, también llamados métodos, nos permiten dividir el trabajo que hace un programa, en tareas más pequeñas separadas de la parte principal. Una función es una subrutina que siempre devuelve el mismo valor dadas las mismas entradas y no tiene efectos secundarios. Las funciones se usan para resolver algún cálculo y devolver el resultado.

Definición de Funciones

Las funciones se definen usando la orden DEF FN, y justo a continuación de la palabra FN se indica el nombre que tendrá la función, los parámetros que aceptará esa función (en nuestro caso, un único número, que hemos llamado X, y que es el número cuyo cuadrado queremos elevar), y finamente un signo = y la expresión matemática que indica cómo calcular esa función.

Para definir una función, la sintaxis de la declaración es la siguiente:

[ Modificadores] tipo_retorno nombre_funcion ([ argumentos])
{
bloque_código;
}

Por ejemplo, en el lenguaje de programación BASIC, podríamos definir una función que nos devuelve el cuadrado de un cierto número de la siguiente manera:

10 DEF FNcuadrado (x) = x * x
20 PRINT "El cuadrado de 2 es "; FNcuadrado(2)

Procedimientos

Un procedimiento en programación es una fracción de un código que realiza o cumple una función específica. Esta parte del código, también puede verse como un subprograma, pudiendo tener sus propias variables y manejo exclusivo de ellas. Los procedimientos se utilizan para realizar una tarea concreta, pero que no involucre devolver nada al programa o subprograma que lo llamó.

Definición de Procedimientos

Los procedimientos se definen de una forma similar a las funciones. Se usa la orden DEF PROC, seguido del nombre del procedimiento y los parámetros que aceptará. A diferencia de las funciones, los procedimientos no devuelven un valor.

Por ejemplo, en el lenguaje de programación BASIC, podríamos definir un procedimiento que imprime un texto centrado en una fila de la pantalla de la siguiente manera:

SUB centrar_texto (texto)
PRINT " "; texto
END SUB
