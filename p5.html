Introducción a la Programación orientada a objetos

Fecha: 11/01/2024


Introducción

La programación orientada a objetos (POO) es un paradigma de programación que utiliza “objetos” para diseñar aplicaciones y programas. Estos objetos se basan en “clases”, que son esencialmente plantillas para crear objetos.

Clases

Una clase es como un plano para un objeto. Define un conjunto de atributos (variables) y métodos (funciones) que caracterizan cualquier objeto de la clase. Los atributos representan las propiedades del objeto, como el color de un coche, mientras que los métodos representan las acciones que puede realizar, como acelerar.

Objetos

Un objeto es una instancia de una clase. Cuando se crea un objeto, se utiliza la definición de la clase para crear ese objeto. Cada objeto tiene su propio conjunto de atributos, que se definen en la clase.

Herencia

La herencia es un principio de la POO que permite a una clase heredar atributos y métodos de otra clase. Por ejemplo, si tienes una clase “Vehículo” y una clase “Coche”, puedes hacer que la clase “Coche” herede atributos de la clase “Vehículo”, como “color” o “modelo”. Esto permite reutilizar el código y hace que el código sea más fácil de mantener.

Polimorfismo

El polimorfismo es otra característica fundamental de la POO. Permite que un objeto se comporte de diferentes maneras según el contexto. Por ejemplo, podrías tener un método en tu clase “Coche” llamado “sonidoDelMotor” que haga un sonido diferente dependiendo de si el coche es un sedán, un deportivo o un SUV.

Más detalles

La POO se basa en dividir el sistema en componentes que contienen operaciones y datos. Cada componente se denomina objeto. Un objeto es una unidad que contiene datos y operaciones que operan sobre esos datos12. Los objetos de un sistema se comunican entre sí mediante mensajes.

Los lenguajes procedimentales están orientados a acciones, y consideran que los datos existen para apoyar las acciones que las funciones realizan. En cambio, los lenguajes orientados a objetos, tienen como unidad fundamental la clase. En una clase, se encapsulan tanto los datos, como las acciones que se realizarán con los datos.

La herencia permite que se puedan definir nuevas clases basadas de unas ya existentes a fin de reutilizar el código, generando así una jerarquía de clases dentro de una aplicación. Si una clase deriva de otra, esta hereda sus atributos y métodos y puede añadir nuevos atributos, métodos o redefinir los heredados.

El polimorfismo es una relajación del sistema de tipos, de tal manera que una referencia a una clase (atributo, parámetro o declaración local o elemento de un vector) acepta direcciones de objetos de dicha clase y de sus clases derivadas (hijas, nietas, …).
